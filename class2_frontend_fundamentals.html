04

you see on screen
has a line of code

05

step 3

Kill the JavaScript.

DevTools → ⋮ → Settings → Preferences → Debugger → Disable JavaScript → Reload

What breaks
Dropdowns freeze
Modals won't open
Dynamic content gone
Buttons dead
What survives
Text still shows
Images still load
Links still work
Structure intact
06

Without JavaScript,

the page is dead.

JS is what makes a website respond to you.

07

step 4

Kill the CSS.

Elements panel → find <link rel="stylesheet"> → delete them. Or use Web Developer extension.

Same content. Same HTML. But now it looks like...
a 1999 document.
No color. No layout. No fonts. Just raw text.
08

Without CSS,

the page is ugly.

CSS is what makes a website look like a website.

09

so here's the truth

Every website is three layers.

JavaScript
Behavior — what the page does
CSS
Style — what the page looks like
HTML
Structure — what the page contains

Strip them one by one. You just saw it happen.

Gmail
Facebook
Figma
Notion
12

the problem

As the UI grows,
raw JavaScript becomes
impossible to manage.

Messy code. Hard to reuse. Painful to work in teams.

13

the solution · 2013

React

Built by Facebook. Because Facebook's own UI became unmanageable.

The idea
Components
Build small, reusable UI pieces. Use them everywhere.
Think of it as
Lego blocks
Navbar, Card, Button — each is one block you build once.

Runs in the browser. Built on top of HTML, CSS, and JS — not a replacement.

14

a page in react is just...

Page
<Navbar /> <Sidebar /> <Feed />
Feed is made of
<PostCard /> <PostCard /> <PostCard />
PostCard is made of
<Avatar /> <LikeButton /> <CommentBox />

Components all the way down.

15

but React has a problem

Everything renders
in the browser.
Slow first load. Bad SEO.

16

the solution to React's problem

Next.js

Wraps React. Adds the missing pieces.

⚡
Server Rendering
Page built on server, not browser
📁
File Routing
Folders = pages. No config.
🔍
SEO Ready
Google can read your content

React is the engine. Next.js is the full car.

19

option 1 · simplest possible

Plain HTML / CSS / JS

project/
├── index.html
├── style.css
└── script.js
To start
Create files.
Open index.html in browser.
No install
No terminal.
No setup. Just files.
20

option 2 · react with vite

React

my-app/
├── src/
│ ├── App.jsx
│ └── main.jsx
├── index.html
├── package.json
└── vite.config.js
Initialize
npm create vite@latest
→ choose React
Run
npm install
npm run dev
localhost:5173
21

option 3 · next.js

Next.js

my-app/
├── app/
│ ├── page.jsx ← /
│ ├── about/
│ │ └── page.jsx ← /about
│ └── layout.jsx
├── public/
├── package.json
└── next.config.js
Initialize
npx create-next-app@latest
Run
npm run dev
localhost:3000
📁 Folders = Routes
No routing config needed.
22

side by side

HTML/CSS/JS React Next.js

Best for Static pages Complex UIs Production apps
